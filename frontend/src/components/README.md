# Components Structure

This directory contains all React components organized by their purpose.

## Directory Structure

```
components/
├── screens/              # Page-level components (full screens)
│   ├── ModelsGrid/      # Models listing screen
│   │   └── index.tsx
│   └── Dashboard/       # Dashboard screen
│       └── index.tsx
│
├── shared/              # Reusable shared components
│   ├── MMDContentCard/  # Generic card for models/stages
│   │   └── index.tsx
│   ├── Sidebar/         # Application sidebar
│   │   └── index.tsx
│   └── MainContent/     # Main content wrapper
│       └── index.tsx
│
└── ui/                  # shadcn/ui components
    ├── avatar.tsx
    ├── badge.tsx
    ├── button.tsx
    ├── card.tsx
    ├── select.tsx
    └── table.tsx
```

## Guidelines

### Screens (`screens/`)
- **Purpose**: Full page/screen components
- **Naming**: Use descriptive names (e.g., `ModelsGrid`, `Dashboard`)
- **Structure**: Each screen in its own folder with `index.tsx`
- **Local Components**: Can include non-exported local components in the same folder
- **Example**:
  ```tsx
  // screens/ModelsGrid/index.tsx
  export function ModelsGrid() { ... }

  // screens/ModelsGrid/ModelFilters.tsx (local, not exported)
  function ModelFilters() { ... }
  ```

### Shared (`shared/`)
- **Purpose**: Reusable components used across multiple screens
- **Naming**: Use generic, reusable names (e.g., `MMDContentCard`, `Sidebar`)
- **Structure**: Each component in its own folder with `index.tsx`
- **Local Components**: Can include helper components in the same folder
- **Example**:
  ```tsx
  // shared/MMDContentCard/index.tsx
  export function MMDContentCard() { ... }

  // shared/MMDContentCard/CarouselDots.tsx (local helper)
  function CarouselDots() { ... }
  ```

### UI (`ui/`)
- **Purpose**: shadcn/ui base components
- **Naming**: Lowercase with hyphens (shadcn convention)
- **Structure**: Flat structure, one component per file
- **Note**: Auto-generated by shadcn CLI, don't manually modify structure

## Import Examples

```tsx
// From App.tsx
import { Sidebar } from "@/components/shared/Sidebar";
import { MainContent } from "@/components/shared/MainContent";

// From MainContent
import { ModelsGrid } from "@/components/screens/ModelsGrid";

// From ModelsGrid
import { MMDContentCard } from "@/components/shared/MMDContentCard";
import { Button } from "@/components/ui/button";
```

## Benefits

1. **Clear Separation**: Screens vs Shared components are clearly distinguished
2. **Encapsulation**: Each component folder can contain local helpers
3. **Scalability**: Easy to add new screens or shared components
4. **Maintainability**: Related files are grouped together
5. **Discoverability**: Intuitive structure makes finding components easy
